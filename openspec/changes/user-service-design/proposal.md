## Why

当前系统使用前端本地选择器和 `X-Role` / `X-User` header 模拟身份，已经足够支持原型验证，但无法支撑生产登录、真实用户目录、跨项目成员管理、审计追踪，也无法安全接入企业已有 SSO/LDAP/OIDC 用户系统。

需要把用户系统抽成独立微服务，并通过清晰的身份适配层与项目管理域解耦，使当前产品可以先使用内置用户服务，后续也能低成本切换或桥接外部用户系统。

## What Changes

- 新增独立用户身份微服务，负责用户、组织/工作区成员、角色、登录会话、令牌签发、外部身份映射和审计事件。
- 项目管理后端不再信任浏览器直接传入的 `X-User` / `X-Role` 作为生产身份来源，改为验证用户服务签发的访问令牌并读取标准化身份声明。
- 增加身份提供方适配接口，支持内置账号、企业 OIDC/OAuth2、LDAP/AD 或其他外部用户系统的后续接入。
- 保留开发环境身份模拟能力，但必须显式标记为 development-only，且不能在生产配置中启用。
- 调整现有工作区访问控制需求，使权限判断基于可信身份声明、工作区成员关系和项目参与关系。
- 不在本次变更中实现完整企业 SSO，只定义可替换架构、API 合同和最小内置用户服务边界。

## Capabilities

### New Capabilities
- `user-identity-service`: 独立用户服务的账户、登录、令牌、用户目录、成员关系、角色分配、外部身份映射和服务间集成合同。

### Modified Capabilities
- `workspace-access-and-notifications`: 将生产身份来源从 `X-Role` / `X-User` header 调整为可信身份声明，并保留开发专用身份模拟约束。

## Impact

- 后端：新增 `user-service` 微服务边界、身份验证中间件、JWT/JWKS 或 introspection 验证、服务间用户查询客户端。
- 前端：新增登录态处理、当前用户菜单、成员选择从用户目录加载；移除生产环境 role/user 手动选择器。
- 数据库：新增用户服务独立 MySQL schema/table；项目域保留 owner/participant 引用但改为稳定 user id。
- 配置：新增内置身份提供方与外部身份提供方配置；生产环境禁止 header spoofing。
- 测试：新增用户服务单元/集成测试、项目服务鉴权集成测试、登录到业务操作的 E2E 测试。
