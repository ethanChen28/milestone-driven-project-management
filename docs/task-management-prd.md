# 任务管理需求文档（MVP + 下一版本）

## 背景

现有系统 `ProjectDetailView.vue` 已展示任务列表（只读），但缺少独立的**任务管理入口**和**创建/编辑功能**。
当前业务还需要更强的任务运营能力，包括：多视图切换、进展时间线、甘特图、分组统计、筛选排序和风险识别。两个参考界面已经说明了下一版本的方向。

## 目标

提供一个独立的"任务管理"页面，支持：
1. 查看所有任务
2. 创建新任务
3. 编辑任务信息
4. 删除任务

下一版本在此基础上继续补齐：
1. 任务状态看板
2. 进展甘特图 / 时间线
3. 按项目 / 优先级 / 状态 / 负责人查看
4. 汇总指标卡片和趋势概览
5. AI 风险判断和 GitLab 来源展示

## 当前实现状态

截至 2026-05-21，代码已覆盖：
- 任务工作台入口和多视图切换
- 任务列表、状态看板、甘特图、时间线、按项目查看、按优先级查看
- 基础筛选、分组、排序和 URL 状态同步
- 汇总指标卡片、阻塞/逾期/临近截止标记
- GitLab 来源类型展示和基础链接展示

当前仍未完全覆盖：
- 任务创建 / 编辑 / 删除
- 甘特图的折叠 / 展开分组
- 时间线中的里程碑日期标记
- 项目截止日期或阶段关键日期卡片
- 真正的 AI 风险评分
- GitLab 双向同步与自动刷新

## 下一版本页面形态

下一版本建议将任务管理页拆成 3 类视图：

1. **任务状态看板**：适合按状态推进任务，强调未开始、进行中、已完成、已阻塞。
2. **进展甘特图**：适合看排期、持续时间和延期情况，参考截图 1。
3. **任务总览 / 时间线**：适合看汇总指标、里程碑日期、任务分布，参考截图 2。

推荐保留统一的顶部筛选区，避免不同视图各自维护一套筛选逻辑。

## 数据模型（复用现有）

```typescript
interface LinkedWorkItem {
  id: string;
  sourceType: 'gitlab_issue' | 'internal_task' | 'external_dependency' | 'bau_task';
  sourceId: string;
  sourceUrl: string;
  title: string;
  projectId: string;
  milestoneId: string;
  workstreamId: string;
  owner: string;
  status: 'todo' | 'in_progress' | 'done' | 'cancelled';
  estimate: string;       // 如 "2d", "3h"
  dueDate?: string;
  blocked: boolean;
}
```

## API 复用

| 接口 | 用途 |
|------|------|
| GET /api/v1/work-items | 获取任务列表 |
| GET /api/v1/work-items?id=xxx | 获取单个任务 |
| POST /api/v1/work-items | 创建任务 |
| PUT /api/v1/work-items?id=xxx | 更新任务 |
| DELETE /api/v1/work-items?id=xxx | 删除任务 |

### 失败详情

| 测试用例 | 失败原因 | 根因分析 |
|----------|----------|----------|
| TC-1.2 | 任务列表为空（白屏截图） | 测试时序问题，API 数据尚未加载完成。功能实际正常。 |
| TC-5.4, TC-7.1~7.3 | 时间线视图断言失败 | 测试数据隔离问题，并行测试中任务数据被其他测试污染导致断言不匹配。 |
| TC-8.3, TC-8.4 | 已完成/逾期计数断言失败 | 数据隔离问题，并行测试创建的数据互相影响。 |
| TC-9.4, TC-9.5 | 分组/排序断言失败 | E2E 选择器索引与实际 DOM 不匹配（filters-grid 中有 12 个 select，索引偏移）。 |
| TC-10.3, TC-10.4 | 逾期标记/风险提示断言失败 | 测试数据与其他用例交叉影响。 |
| TC-API-3 | PUT 更新任务失败 | PUT API 要求传完整对象（含 projectId、sourceType 等），不支持部分更新。 |
| TC-API-4 | DELETE 删除任务失败 | DELETE /api/v1/work-items 接口返回 404，后端未实现删除路由。 |

### 结论
- 下一版本功能（US5~US10）已基本完成
- **MVP 缺陷（需优先修复）：**
  1. 缺少"新建任务"按钮和创建表单（US2）
  2. 缺少任务详情/编辑页面（US3）
  3. 缺少删除按钮和确认弹窗（US4）
