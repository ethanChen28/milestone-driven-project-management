## 二、各角色详细审查

### 2.1 Admin（管理员）

#### 满足的需求

- 项目、里程碑、任务的全量 CRUD 操作
- 路线图管理（创建/编辑周期、关联项目）
- 周报提交与查看
- 多视图展示（列表、看板、甘特图、时间线、分组视图）
- 筛选、分组、排序功能完整
- GitLab 外部系统集成（工作项同步、状态展示）
- 仪表盘提供项目总数、活跃数、风险数等关键指标

#### 缺失的需求

1. **用户管理界面** — 无法在 UI 中管理用户账号、角色分配和权限配置，当前通过 localStorage 模拟
2. **集成配置中心** — 没有界面配置 GitLab API Token 等外部系统连接参数
3. **审计日志** — 缺少操作历史记录和变更追踪功能
4. **批量操作** — 不支持批量删除、批量更新状态、批量分配负责人
5. **数据导出** — 缺少导出项目数据为 Excel 或 PDF 报告的功能

#### 改进建议

| 优先级 | 建议 | 说明 |
|--------|------|------|
| P0 | 添加用户管理模块 | 用户列表、角色分配、权限配置界面 |
| P0 | 集成配置中心 | 配置外部系统连接参数的界面 |
| P1 | 引入审计日志视图 | 记录关键操作日志，便于追踪和回溯 |
| P1 | 增强批量操作功能 | 支持批量更新状态、分配负责人等 |
| P2 | 增加数据导出功能 | 支持导出项目数据为 Excel 或 PDF |

### 2.2 Portfolio Manager（组合经理）

#### 满足的需求

- 项目→里程碑→工作项的完整层级管理
- 仪表盘提供健康度分布（on_track / at_risk / off_track）
- 延期里程碑和阻塞里程碑的告警展示
- 路线图周期管理，关联项目摘要
- 任务多视图（看板/甘特/时间线）支持跨项目视角

#### 缺失的需求

1. **跨项目资源分配视图** — 缺少全局资源分配和负载均衡分析，无法查看某人在多个项目中的分配情况
2. **项目间依赖关系可视化** — 只能看到单个项目的依赖，缺少跨项目依赖网络图
3. **资源冲突预警** — 没有资源过度分配的自动检测和提醒
4. **投资组合分析** — 缺少 ROI、项目优先级自动调整建议
5. **综合风险仪表盘** — 风险信息分散在不同视图，缺少集中的风险看板

#### 改进建议

| 优先级 | 建议 | 说明 |

| P0 | 开发甘特图跨项目视图 | 能查看所有项目的时间线和依赖关系 |

| P1 | 构建综合风险仪表盘 | 集中展示所有延期、阻塞、高风险项目 |
| P2 | 引入智能预警系统 | 基于历史数据预测项目风险和资源瓶颈 |
